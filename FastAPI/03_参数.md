同一段接口逻辑，根据参数不同返回不同的数据
## 参数
参数就是客户端发送请求时附带的额外信息和指令

参数的作用是让同一个接口能根据不同的输入，返回不同的输出，实现动态交互

### 路径参数
```python
@app.get("/book/{id}")
async def get_book(id:int):
  return {"id":id,"title":{"这是第{id}本书"}
```
路径参数--类型注解Path
导入FastAPI的Path函数
```python
@app.get("/book/{id}")
async def get_book(id:int=Path(...,gt=0,lt=101,description='书籍id,取值范围1-100')):
  return {"id":id,"title":{"这是第{id}本书"}
```
需求：查找书籍的作者，路径参数name，长度范围2-10
```python
@app.get("{/author/{name}}")
async def get_name(name:str=Path(...,min_length=2,max_length=10):
  return {"msg":f'这是{name}的信息'}
```

**路径参数出现在什么位置？**

URL路径的一部分 

**如何为路径参数添加类型注解？**

Python原生注解和Path注解

### 查询参数
声明的参数不是路径参数，路径操作函数会把该参数自动解释为查询参数

需求 查询新闻->分页，skip：跳过的记录数 limit:返回的记录数 10
```Python
@app.get("/news/news_list")
async def get_news_list(skip:int,limit:int=10)：
  return {"skip":skip,"limit":limit}
```
查询参数额外的Query注解
```Python
import fastapi import Query
@app.get("/news/news_list")
async def get_news_list(
  skip:int=Query(0,description="跳过的记录数"，lt=100),
  limit:int=Query(10,description="返回的记录数")
)：
  return {"skip":skip,"limit":limit}
```
**查询参数出现在什么位置？**

URL？之后，k1=v1&k2=v2

**如何为查询参数添加类型注释？**

Python原生注释和Query注释

### 请求体参数
在HTTP协议中，一个完整的请求由三部分组成：

请求行：包含方法、URL、协议版本

请求头：元数据信息

请求体：实际要发送的数据内容

1.定义类型
```Python
from pydantic import BaseModel
class User(BaseModel):
  username:str
  password:str 
```
2.类型注解
```Python
app.post("/register")
async def register(user:User):
  return user
```
请求体参数--类型注解
导入pydantic的Field函数
```Python
from pydantic import BaseModel,Field
class User(BaseModel):
  username:str=Field(default="张三"，min_length=2,max_length=10,description="用户名，长度要求2-10个字)
  password:str=Field(min_length=3,max_length=20)
```
请求体参数的作用：创建、更新资源
