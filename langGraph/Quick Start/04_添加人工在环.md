# LangGraph 人工在环（Human-in-the-Loop）知识点总结

本文档记录了在 LangGraph 中实现人工在环（Human-in-the-Loop）的核心概念、实现方法、关键代码及常见问题。通过人工在环，可以让智能体在执行关键操作前暂停并等待人工确认，或主动请求人工协助，从而提高系统的安全性和灵活性。

---

## 一、什么是人工在环？

人工在环是指在自动化流程中引入人工判断的环节。在 LangGraph 中，我们可以在图的任意节点插入中断点，当执行到该节点时，图会暂停并保存状态，等待外部程序提供输入后继续运行。这常用于：

- **工具调用确认**：在执行敏感操作（如数据库写入、发送邮件、调用外部 API）前，请求用户批准。
- **人工辅助**：当模型遇到无法处理的问题时，主动请求人工介入，获取额外信息后继续。

---

## 二、实现方式

LangGraph 提供了两种主要方式来实现人工在环：

### 1. 节点插入法（推荐）
在图中显式添加一个“人工确认节点”，通过条件边将需要确认的流程引导至此节点。节点内部调用 `interrupt()` 暂停，外部程序获取中断信息后，用户输入决策，再通过 `Command(resume=...)` 恢复。

**特点**：
- 所有需要确认的步骤都经过同一个节点，逻辑清晰。
- 不依赖模型主动调用工具，适用于固定规则（如所有工具调用都需确认）。

### 2. 工具封装法
将人工协助封装成一个普通工具，让模型在需要时主动调用。该工具内部调用 `interrupt()`，外部程序同样需要处理中断。

**特点**：
- 模型自主决定何时请求人工帮助，更灵活。
- 需要额外处理中断恢复，且要求模型能正确调用该工具。

---

## 三、核心组件

| 组件 | 作用 |
|------|------|
| `interrupt(value)` | 在节点或工具内部调用，立即暂停图执行，并将 `value` 保存为中断数据。 |
| `Command(resume=data)` | 用于恢复中断，`data` 会成为 `interrupt()` 的返回值。 |
| `checkpointer` | 必须配置检查点存储器（如 `MemorySaver`），才能保存中断状态，使恢复成为可能。 |
| `stream_mode="values"` | 流式输出模式，可捕获 `__interrupt__` 事件。 |

---

## 四、中断事件的格式与安全处理

当 `interrupt()` 被调用时，LangGraph 会在流式输出中产生一个包含 `__interrupt__` 键的事件。不同版本中该值的结构可能不同，可能是：

- 单个 `Interrupt` 对象
- 包含 `Interrupt` 对象的列表
- 元组或嵌套结构

**安全提取中断数据的通用函数**：
```python
def extract_interrupt_data(event):
    interrupt_obj = event.get("__interrupt__")
    # 递归提取实际值
    def extract(obj):
        if isinstance(obj, (list, tuple)) and obj:
            return extract(obj[0])
        if hasattr(obj, "value"):
            return extract(obj.value)
        return obj
    return extract(interrupt_obj)
```

## 五、节点插入法完整示例

下面是一个使用节点插入法实现人工确认的完整示例，包含 Tavily 搜索工具、记忆、中断处理。

```python
import os
from typing import Annotated
from langchain_community.chat_models import ChatTongyi
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import ToolNode, tools_condition
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import interrupt, Command
from langchain_tavily import TavilySearch
from typing_extensions import TypedDict

# 配置
API_KEY = os.getenv("DASHSCOPE_API_KEY")
BASE_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1"
MODEL = "qwen-max"
os.environ["TAVILY_API_KEY"] = "你的Tavily密钥"

# 工具
search = TavilySearch(max_results=2)
tools = [search]
tool_node = ToolNode(tools)

# 状态
class State(TypedDict):
    messages: Annotated[list, add_messages]

# LLM
llm = ChatTongyi(model=MODEL, api_key=API_KEY, api_base=BASE_URL)
llm_with_tools = llm.bind_tools(tools)

# 人工确认节点
def human_approval(state: State):
    messages = state.get("messages", [])
    if not messages:
        return state
    last = messages[-1]
    if not (hasattr(last, "tool_calls") and last.tool_calls):
        return state

    # 提取工具调用信息
    tools_info = [f"{tc['name']}({tc['args']})" for tc in last.tool_calls]
    print(f"\n即将执行工具：{', '.join(tools_info)}")
    # 中断，等待外部输入
    user_decision = interrupt({
        "type": "human_approval",
        "tools": tools_info,
        "message": "是否继续？(yes/no)"
    })

    if user_decision == "yes":
        return state
    else:
        # 用户拒绝：移除 tool_calls 并添加一条消息
        new_messages = list(messages[:-1])
        new_messages.append({
            "role": "assistant",
            "content": "用户取消了工具调用。"
        })
        return {"messages": new_messages}

# 路由函数
def route_after_chatbot(state: State):
    messages = state.get("messages", [])
    if not messages:
        return END
    last = messages[-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "human_approval"
    return END

def route_after_approval(state: State):
    messages = state.get("messages", [])
    if not messages:
        return END
    last = messages[-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "tools"
    return END

# 构建图
builder = StateGraph(State)
builder.add_node("chatbot", lambda state: {"messages": [llm_with_tools.invoke(state["messages"])]})
builder.add_node("human_approval", human_approval)
builder.add_node("tools", tool_node)

builder.add_edge(START, "chatbot")
builder.add_conditional_edges("chatbot", route_after_chatbot, {
    "human_approval": "human_approval",
    END: END
})
builder.add_conditional_edges("human_approval", route_after_approval, {
    "tools": "tools",
    END: END
})
builder.add_edge("tools", "chatbot")

memory = MemorySaver()
graph = builder.compile(checkpointer=memory)

# 流式运行函数（处理中断）
def run_with_interrupts(user_input: str, config: dict):
    for event in graph.stream(
        {"messages": [{"role": "user", "content": user_input}]},
        config=config,
    ):
        if "__interrupt__" in event:
            # 提取中断数据
            interrupt_data = extract_interrupt_data(event)
            # 打印提示
            if isinstance(interrupt_data, dict):
                print(f"\n[系统] {interrupt_data.get('message', '需要确认')}")
            else:
                print(f"\n[系统] {interrupt_data}")
            # 获取用户决策
            decision = input("请输入 yes 或 no: ").strip().lower()
            # 恢复执行
            for resume_event in graph.stream(Command(resume=decision), config=config):
                if "__interrupt__" in resume_event:
                    continue
                for v in resume_event.values():
                    if "messages" in v:
                        print("Assistant:", v["messages"][-1].content)
            return
        else:
            for v in event.values():
                if "messages" in v:
                    print("Assistant:", v["messages"][-1].content)

# 交互
config = {"configurable": {"thread_id": "1"}}
while True:
    try:
        user_input = input("User: ")
        if user_input.lower() in ["quit", "exit", "q"]:
            break
        run_with_interrupts(user_input, config)
    except Exception as e:
        print(f"错误: {e}")
        break
```

## 六、工具封装法示例（模型主动请求人工）

```python
from langchain_core.tools import tool

@tool
def human_assistance(query: str) -> str:
    """当需要人工帮助时调用此工具"""
    response = interrupt({"query": query})
    return response["data"]

tools = [search, human_assistance]  # 将人工工具加入列表
# 其余构建流程同前，但不需要显式的 human_approval 节点
```

**注意**：使用工具封装法时，外部恢复必须传入符合工具期望的数据结构（例如 `{"data": answer}`），并且模型必须有能力在适当时候调用该工具。

## 七、常见问题及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `'tuple' object has no attribute 'get'` | 中断事件结构为元组或 Interrupt 对象，直接调用 `.get()` 失败 | 使用递归提取函数安全获取数据 |
| 恢复后图不继续执行 | 恢复时传入的 `resume` 数据格式与 `interrupt` 期望不匹配 | 检查 `interrupt` 期望的返回类型，例如工具内期望 `response["data"]`，则恢复需传入 `Command(resume={"data": answer})` |
| 中断事件未被捕获 | `stream` 时未启用 `stream_mode="values"`，或未正确处理 `__interrupt__` 键 | 确保使用 `graph.stream(..., config, stream_mode="values")`，并在循环中检测 `"__interrupt__" in event` |
| 模型多次调用工具 | 模型一次性生成多个 `tool_calls` | 使用 `ToolNode` 会自动按顺序执行多个工具；也可在模型中设置 `tool_choice` 限制 |
| 记忆失效 | 未正确传入 `config` 或 `thread_id` | 在 `stream` 和 `invoke` 中必须传入 `config`，且 `thread_id` 一致 |

---

## 八、记忆与中断的配合

- `checkpointer`（如 `MemorySaver`）会保存中断时的完整状态，因此即使程序重启，只要使用相同的 `thread_id`，中断可以恢复。
- 恢复时通过 `Command(resume=...)` 继续，图会从断点接着执行。
- 这对于长时间等待的人工审批流程非常有用。

---

## 九、总结

人工在环是构建可靠智能体的重要机制。LangGraph 通过 `interrupt` 和 `Command` 提供了灵活的中断与恢复能力。根据场景选择节点插入法或工具封装法，并注意正确处理中断事件的数据格式，即可实现安全可控的人工介入流程。
