# LangGraph 聊天机器人开发笔记：API 认证与配置问题总结

本文档记录了在使用 LangGraph + 阿里云百炼（DashScope）搭建聊天机器人时遇到的 API 认证问题及解决方案，并整理了相关知识点，便于后续查阅和避免重复踩坑。

---

## 一、问题描述

### 错误信息
```
openai.AuthenticationError: Error code: 401 - {'error': {'message': 'Incorrect API key provided...', 'type': 'invalid_request_error', 'code': 'invalid_api_key'}}
```

### 错误原因
- **API Key 无效**：使用了错误的 API Key 或者将 Base URL 误填为 API Key。
- **缺少 Base URL**：使用了阿里云百炼的 OpenAI 兼容模式，但未设置正确的 `OPENAI_API_BASE`，导致请求被发送到 OpenAI 官方服务器，而密钥却是阿里云的，认证失败。

---

## 二、解决方案

### 1. 正确配置阿里云百炼 OpenAI 兼容模式
阿里云百炼支持 OpenAI SDK 调用，需同时设置 **API Key** 和 **Base URL**。

#### 方法一：通过环境变量（推荐生产环境）
```python
import os

os.environ["OPENAI_API_KEY"] = "sk-你的阿里云百炼密钥"          # 真实密钥
os.environ["OPENAI_API_BASE"] = "https://dashscope.aliyuncs.com/compatible-mode/v1"

# 之后 LangChain 会自动读取这些环境变量
from langchain_classic.chat_models import init_chat_model
llm = init_chat_model("openai:qwen-max")  # 或 qwen-plus, qwen-turbo
```

#### 方法二：直接硬编码（仅用于本地测试）
```python
from langchain_classic.chat_models import ChatOpenAI

llm = ChatOpenAI(
    model="qwen-max",
    openai_api_key="sk-你的阿里云百炼密钥",
    openai_api_base="https://dashscope.aliyuncs.com/compatible-mode/v1",
)
```

> ⚠️ **安全提醒**：硬编码密钥仅限本地调试，**切勿提交到公开代码仓库**。正式环境请使用环境变量或密钥管理服务。

## 三、关键语法与知识点

### 1. LangGraph 状态定义
使用 `TypedDict` 定义状态，`add_messages` 用于自动合并消息列表。
```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]   # 注意键名是 messages，不是 massages
```

### 2. 构建 StateGraph
- 创建图对象：`graph_builder = StateGraph(State)`
- 添加节点：`graph_builder.add_node("chatbot", chatbot)`
- 添加边：`graph_builder.add_edge(START, "chatbot")`
- 编译：`graph = graph_builder.compile()`

### 3. 模型初始化
- 使用 `init_chat_model("openai:qwen-max")` 时，模型名格式为 `provider:model_name`。
- 直接使用 `ChatOpenAI` 更灵活，可以显式传入 `openai_api_key` 和 `openai_api_base`。

### 4. 流式输出处理
`graph.stream()` 返回生成器，遍历每个事件，提取最后一条助手消息。
```python
for event in graph.stream({"messages": [{"role": "user", "content": user_input}]}):
    for value in event.values():
        print("Assistant:", value["messages"][-1].content)
```

### 5. 异常处理
避免使用裸 `except:`，应捕获具体异常并打印错误信息。
```python
try:
    ...
except Exception as e:
    print(f"错误: {e}")
    break
```

## 五、完整可用代码（硬编码版，本地测试用）

```python
import os
from typing import Annotated
from langchain_classic.chat_models import ChatOpenAI
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START
from langgraph.graph.message import add_messages

# ========== 阿里云百炼配置 ==========
API_KEY = "sk-你的真实密钥"                     # 替换为实际密钥
BASE_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1"  # 根据地域调整
MODEL = "qwen-max"                            # 可选 qwen-plus, qwen-turbo
# ===================================

class State(TypedDict):
    messages: Annotated[list, add_messages]

# 创建 LLM 实例
llm = ChatOpenAI(
    model=MODEL,
    openai_api_key=API_KEY,
    openai_api_base=BASE_URL,
)

graph_builder = StateGraph(State)

def chatbot(state: State):
    return {"messages": [llm.invoke(state["messages"])]}

graph_builder.add_node("chatbot", chatbot)
graph_builder.add_edge(START, "chatbot")
graph = graph_builder.compile()

def stream_graph_updates(user_input: str):
    for event in graph.stream({"messages": [{"role": "user", "content": user_input}]}):
        for value in event.values():
            print("Assistant:", value["messages"][-1].content)

while True:
    try:
        user_input = input("User: ")
        if user_input.lower() in ["quit", "exit", "q"]:
            print("Goodbye!")
            break
        stream_graph_updates(user_input)
    except Exception as e:
        print(f"错误: {e}")
        break
```

## 六、总结与避坑指南

1. **区分 OpenAI 官方与兼容服务**  
   - 官方 OpenAI：只需 `OPENAI_API_KEY`，无需设置 Base URL。  
   - 阿里云百炼：必须同时设置 `OPENAI_API_KEY` 和 `OPENAI_API_BASE`。

2. **环境变量设置方式**  
   - 系统级（终端）：`export OPENAI_API_KEY=...`  
   - 代码级：`os.environ["OPENAI_API_KEY"] = "..."`（仅在当前进程生效）

3. **LangGraph 状态键名**  
   - 使用 `messages`（复数），避免拼写错误导致无法传递消息。

4. **异常处理**  
   - 始终打印具体异常信息，便于定位问题。

5. **安全性**  
   - 硬编码密钥仅用于快速测试，生产环境务必使用环境变量或密钥管理工具。
