# LangGraph 记忆（Memory）模块知识点总结

本文档记录了在 LangGraph 中为聊天机器人添加**记忆功能**（会话持久化）的核心概念、实现方法和关键细节。通过记忆功能，智能体可以记住多轮对话的历史，实现连贯的交互体验。

---

## 一、为什么需要记忆？

在默认情况下，LangGraph 的图是无状态的——每次调用 `graph.stream()` 或 `graph.invoke()` 都是独立的，不会自动保留之前的对话历史。如果我们希望智能体能记住用户之前说过的话（例如多轮问答、上下文引用），就需要**手动管理状态**或使用内置的**检查点（Checkpointer）**机制。

LangGraph 提供的 **MemorySaver** 是一个轻量级的内存检查点存储器，可以在运行时自动保存和恢复图的状态。

---

## 二、核心概念

### 1. Checkpointer（检查点）
- 在 LangGraph 中，一个“检查点”是图在某个执行步骤时的**完整状态快照**。
- 通过 `checkpointer` 参数，图会在每个节点执行后自动保存状态。
- 下次调用时，通过提供相同的 `thread_id`，图会自动从最新的检查点恢复状态，从而实现记忆。

### 2. MemorySaver
- 这是 LangGraph 内置的一个简单检查点存储器，将状态保存在**内存**中。
- 适用于开发测试、单进程场景。
- 生产环境可替换为其他持久化存储（如 SQLite、Redis、PostgreSQL 等），但使用方法一致。

### 3. thread_id（线程ID）
- `thread_id` 是会话的唯一标识符，用于区分不同用户的对话或同一用户的不同会话。
- 在调用 `stream` / `invoke` 时通过 `config` 参数传递。
- 相同的 `thread_id` 会共享同一份记忆，不同的 `thread_id` 记忆相互隔离。

---

## 三、如何添加记忆

### 步骤 1：导入 MemorySaver
```python
from langgraph.checkpoint.memory import MemorySaver
```

### 步骤 2：创建 MemorySaver 实例
```python
memory = MemorySaver()
```

### 步骤 3：编译图时传入 checkpointer
```python
graph = graph_builder.compile(checkpointer=memory)
```

### 步骤 4：在调用时传递 config（包含 thread_id）
```python
config = {"configurable": {"thread_id": "user_123"}}

# 在流式或普通调用中传入 config
for event in graph.stream(input_data, config=config):
    # 处理事件
    pass
```

## 四、完整代码示例

下面是一个简单的 LangGraph 聊天机器人，带有记忆功能（省略了工具和路由的细节，仅展示记忆部分）：

```python
import os
from typing import Annotated
from typing_extensions import TypedDict
from langchain_community.chat_models import ChatTongyi
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.checkpoint.memory import MemorySaver

# 定义状态
class State(TypedDict):
    messages: Annotated[list, add_messages]

# 初始化 LLM
llm = ChatTongyi(model="qwen-max", api_key=os.getenv("DASHSCOPE_API_KEY"))

# 定义聊天节点
def chatbot(state: State):
    return {"messages": [llm.invoke(state["messages"])]}

# 构建图
graph_builder = StateGraph(State)
graph_builder.add_node("chatbot", chatbot)
graph_builder.add_edge(START, "chatbot")
graph_builder.add_edge("chatbot", END)

# 添加记忆
memory = MemorySaver()
graph = graph_builder.compile(checkpointer=memory)

# 使用记忆进行多轮对话
config = {"configurable": {"thread_id": "user_123"}}

def chat_with_memory(user_input: str):
    for event in graph.stream(
        {"messages": [{"role": "user", "content": user_input}]},
        config=config
    ):
        for value in event.values():
            print("Assistant:", value["messages"][-1].content)

# 第一轮
chat_with_memory("我叫小明")
# 第二轮（会记住之前的内容）
chat_with_memory("我叫什么名字？")   # 输出：你叫小明
```

## 五、关键知识点

### 1. 状态保存的时机
- 图会在**每个节点执行后**自动保存检查点（前提是编译时传入了 `checkpointer`）。
- 保存的内容包括整个 `State` 字典。

### 2. 恢复状态的机制
- 当使用相同的 `thread_id` 再次调用时，LangGraph 会**自动加载**该线程最近一次保存的检查点作为初始状态。
- 因此你无需在代码中手动读取或合并历史消息，`messages` 列表会自动延续。

### 3. 配置参数详解
```python
config = {"configurable": {"thread_id": "唯一标识符"}}
```
- `thread_id` 是必须的，用于定位检查点。
- 还可以传递其他自定义配置，但 `configurable` 下的 `thread_id` 是 LangGraph 约定用于区分会话的键。

### 4. 多线程与并发
- 如果同时处理多个用户的请求，只需为每个用户分配不同的 `thread_id`，检查点会自动隔离。
- `MemorySaver` 是线程安全的，但仅限内存存储，适合开发环境。

### 5. 持久化存储
- 若需要生产环境持久化，可替换 `MemorySaver` 为其他 `BaseCheckpointSaver` 实现，例如：
  ```python
  from langgraph.checkpoint.sqlite import SqliteSaver
  memory = SqliteSaver.from_conn_string("checkpoints.db")
  ```
  使用方法完全一致，只需修改导入和实例化部分。

### 6. 注意事项
- **内存占用**：`MemorySaver` 将所有会话状态存储在 RAM 中，长时间运行可能导致内存溢出，仅限测试。
- **线程 ID 管理**：你需要自己维护 `thread_id` 与用户/会话的映射关系。
- **版本兼容**：不同 LangGraph 版本 API 可能略有差异，建议参考官方文档。

## 六、常见问题

### Q1：为什么我加了 memory 但对话还是记不住？
- 检查是否在 `compile` 时传入了 `checkpointer=memory`。
- 检查调用 `stream` / `invoke` 时是否传入了 `config` 且包含 `thread_id`。
- 检查 `thread_id` 是否每次调用都相同。

### Q2：MemorySaver 和 add_messages 的关系？
- `add_messages` 注解负责**合并**新消息到现有列表，确保消息历史正确累积。
- `MemorySaver` 负责**保存**整个状态（包括消息列表），两者协同工作。

### Q3：如何清空某个会话的记忆？
- 只需在下次调用时更换一个新的 `thread_id`，旧会话的检查点将不再被加载（但依然占用内存直到进程结束）。
- 若想主动删除，可以调用检查点存储器的删除方法（具体实现不同）。

---

## 七、总结

| 组件 | 作用 |
|------|------|
| `MemorySaver` | 内存检查点存储器，用于保存和恢复图的状态 |
| `checkpointer` | 编译图的参数，启用自动检查点功能 |
| `config` | 每次调用时传递的配置字典，需包含 `thread_id` |
| `thread_id` | 会话唯一标识，相同 ID 共享记忆 |

通过这三步（导入 MemorySaver → 编译时传入 → 调用时传 config），你就能轻松为 LangGraph 聊天机器人添加多轮对话记忆功能。
