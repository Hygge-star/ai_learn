# LangGraph 添加工具（Tool）知识点总结

本文档记录了在 LangGraph 聊天机器人中集成外部工具（如 Tavily 搜索引擎）的完整流程、关键语法、常见错误及解决方案。通过添加工具，可以让 LLM 在需要时调用外部 API 获取实时信息，从而增强智能体的能力。

---

## 一、为什么需要工具？

LLM（大语言模型）本身的知识是静态的，无法获取实时信息或执行特定操作。通过工具（Tools），我们可以让 LLM 在对话过程中动态决定调用外部函数（如搜索、计算、数据库查询等），并将结果返回给用户。

在 LangGraph 中，工具被封装为 **节点**，并通过 **条件边** 实现智能路由。

---

## 二、整体流程概览

1. **定义工具**：实例化具体的工具（如 `TavilySearch`）。
2. **绑定工具到 LLM**：使用 `bind_tools` 让模型知道可以使用哪些工具。
3. **创建工具节点**：编写一个节点函数，用于执行工具调用并返回结果。
4. **定义路由函数**：根据 LLM 的输出判断是否需要调用工具。
5. **构建图**：
   - 添加 `chatbot` 节点（使用绑定了工具的 LLM）。
   - 添加 `tools` 节点（执行工具）。
   - 添加条件边：从 `chatbot` 根据路由函数决定去 `tools` 还是结束。
   - 添加普通边：从 `tools` 回到 `chatbot`，让模型处理工具返回的结果。

## 三、关键代码与语法

### 1. 导入所需模块
```python
import os
import json
from typing import Annotated
from typing_extensions import TypedDict

from langchain_community.chat_models import ChatTongyi   # 或其他 LLM 类
from langchain_tavily import TavilySearch                # 搜索工具
from langchain_core.messages import ToolMessage          # 工具返回的消息类型

from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
```

### 2. 定义状态
状态包含一个 `messages` 列表，并使用 `add_messages` 注解来自动合并消息。
```python
class State(TypedDict):
    messages: Annotated[list, add_messages]
```

### 3. 初始化工具
```python
# 创建搜索工具实例，设置返回结果数量
tool = TavilySearch(max_results=2)
tools = [tool]                     # 如果有多个工具，放入列表
```

### 4. 绑定工具到 LLM
```python
# 初始化 LLM（这里以阿里云通义千问为例）
llm = ChatTongyi(
    model="qwen-max",
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    api_base="https://dashscope.aliyuncs.com/compatible-mode/v1"
)

# 绑定工具，返回一个新的模型对象
llm_with_tools = llm.bind_tools(tools)
```

### 5. 创建工具节点
工具节点负责执行 LLM 请求的工具调用，并将结果包装成 `ToolMessage` 返回。

```python
class BasicToolNode:
    """一个节点，用于执行最后一条 AIMessage 中请求的工具"""
    def __init__(self, tools: list):
        # 将工具列表转换为字典，方便按名称快速查找
        self.tools_by_name = {tool.name: tool for tool in tools}

    def __call__(self, inputs: dict):
        # 从输入中取出消息列表
        messages = inputs.get("messages", [])
        if not messages:
            raise ValueError("No messages found in input")
        # 取最后一条消息，应该包含 tool_calls
        message = messages[-1]

        outputs = []
        for tool_call in message.tool_calls:
            # 根据工具名称获取对应工具
            tool_result = self.tools_by_name[tool_call["name"]].invoke(tool_call["args"])
            # 构造 ToolMessage
            outputs.append(
                ToolMessage(
                    content=json.dumps(tool_result),   # 将结果转为 JSON 字符串
                    name=tool_call["name"],
                    tool_call_id=tool_call["id"],
                )
            )
        return {"messages": outputs}
```

### 6. 定义路由函数
路由函数根据最后一条 AI 消息是否包含 `tool_calls` 来决定下一步是执行工具还是结束。

```python
def route_tools(state: State):
    """
    返回：
    - "tools" 如果最后一条消息有工具调用
    - END 否则
    """
    messages = state.get("messages", [])
    if not messages:
        raise ValueError("No messages found in input state")

    ai_message = messages[-1]

    # 判断是否存在工具调用
    if hasattr(ai_message, "tool_calls") and len(ai_message.tool_calls) > 0:
        return "tools"
    return END
```

### 7. 构建图
```python
# 创建图构建器
graph_builder = StateGraph(State)

# 添加节点
graph_builder.add_node("chatbot", lambda state: {"messages": [llm_with_tools.invoke(state["messages"])]})
graph_builder.add_node("tools", BasicToolNode(tools))

# 添加边
graph_builder.add_edge(START, "chatbot")                         # 入口
graph_builder.add_conditional_edges("chatbot", route_tools, {"tools": "tools", END: END})  # 条件路由
graph_builder.add_edge("tools", "chatbot")                       # 工具执行后回到 chatbot

# 编译图
graph = graph_builder.compile()
```

### 8. 运行图（流式输出）
```python
def stream_graph_updates(user_input: str):
    for event in graph.stream({"messages": [{"role": "user", "content": user_input}]}):
        for value in event.values():
            print("Assistant:", value["messages"][-1].content)
```

## 四、常见错误与解决方法

### ❌ 错误1：`KeyError: -1` 或 `KeyError: 'messages'`
- **原因**：在路由函数中错误地使用 `state[-1]` 或 `state.get(["messages"])`。`state` 是一个字典，应通过键名访问。
- **解决**：正确写法是 `state.get("messages", [])`，然后取最后一个元素 `messages[-1]`。

### ❌ 错误2：模型从不调用工具
- **原因**：在 `chatbot` 节点中使用了未绑定工具的 LLM（例如 `llm.invoke` 而非 `llm_with_tools.invoke`）。
- **解决**：确保节点中使用的是 `bind_tools` 返回的模型对象。

### ❌ 错误3：`ValueError: No messages found in input`
- **原因**：输入状态中缺少 `messages` 字段，或消息列表为空。
- **解决**：在调用 `graph.stream` 时确保传入 `{"messages": [...]}`，并在节点和路由函数中添加空值检查。

### ❌ 错误4：工具调用后，模型无法正确解析结果
- **原因**：工具返回的结果格式不符合模型期望，或未将结果作为消息追加到状态中。
- **解决**：工具节点必须返回 `{"messages": [ToolMessage(...)]}`，这样 `add_messages` 会自动合并到历史中。

### ❌ 错误5：环境变量未设置
- **原因**：API 密钥未正确配置。
- **解决**：使用 `os.environ["TAVILY_API_KEY"] = "..."` 或系统环境变量设置。注意区分不同服务的密钥。

## 五、关键知识点总结

| 知识点 | 说明 |
|--------|------|
| **工具节点** | 一个函数或可调用类，输入一个字典（包含 `messages`），执行工具调用，返回 `{"messages": [...]}`。 |
| **`bind_tools`** | LLM 对象的方法，将工具列表绑定到模型，使模型在输出中包含 `tool_calls` 字段。 |
| **`ToolMessage`** | LangChain 中用于封装工具返回结果的消息类型，必须包含 `tool_call_id` 与请求匹配。 |
| **条件边** | `add_conditional_edges` 允许根据状态动态选择下一个节点。路由函数返回节点名称或 `END`。 |
| **`add_messages` 注解** | 自动将新消息追加到现有列表，避免手动管理消息历史。 |
| **工具名称匹配** | 工具节点的 `tools_by_name` 字典确保按名称正确调用工具。 |

## 六、完整示例代码

```python
import os
import json
from typing import Annotated
from typing_extensions import TypedDict
from langchain_community.chat_models import ChatTongyi
from langchain_tavily import TavilySearch
from langchain_core.messages import ToolMessage
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

# ---------- 配置 ----------
os.environ["TAVILY_API_KEY"] = "你的 Tavily 密钥"
API_KEY = os.getenv("DASHSCOPE_API_KEY")
BASE_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1"
MODEL = "qwen-max"

# ---------- 工具 ----------
tool = TavilySearch(max_results=2)
tools = [tool]

# ---------- 状态 ----------
class State(TypedDict):
    messages: Annotated[list, add_messages]

# ---------- 工具节点 ----------
class BasicToolNode:
    def __init__(self, tools: list):
        self.tools_by_name = {tool.name: tool for tool in tools}
    def __call__(self, inputs: dict):
        messages = inputs.get("messages", [])
        if not messages:
            raise ValueError("No messages")
        message = messages[-1]
        outputs = []
        for tool_call in message.tool_calls:
            result = self.tools_by_name[tool_call["name"]].invoke(tool_call["args"])
            outputs.append(ToolMessage(
                content=json.dumps(result),
                name=tool_call["name"],
                tool_call_id=tool_call["id"],
            ))
        return {"messages": outputs}

# ---------- 路由函数 ----------
def route_tools(state: State):
    messages = state.get("messages", [])
    if not messages:
        return END
    ai_message = messages[-1]
    if hasattr(ai_message, "tool_calls") and len(ai_message.tool_calls) > 0:
        return "tools"
    return END

# ---------- 构建图 ----------
llm = ChatTongyi(model=MODEL, api_key=API_KEY, api_base=BASE_URL)
llm_with_tools = llm.bind_tools(tools)

graph_builder = StateGraph(State)
graph_builder.add_node("chatbot", lambda state: {"messages": [llm_with_tools.invoke(state["messages"])]})
graph_builder.add_node("tools", BasicToolNode(tools))
graph_builder.add_edge(START, "chatbot")
graph_builder.add_conditional_edges("chatbot", route_tools, {"tools": "tools", END: END})
graph_builder.add_edge("tools", "chatbot")
graph = graph_builder.compile()

# ---------- 运行 ----------
def stream_updates(user_input: str):
    for event in graph.stream({"messages": [{"role": "user", "content": user_input}]}):
        for value in event.values():
            print("Assistant:", value["messages"][-1].content)

while True:
    try:
        user_input = input("User: ")
        if user_input.lower() in ["quit", "exit", "q"]:
            break
        stream_updates(user_input)
    except Exception as e:
        print(f"错误: {e}")
        break
```

## 七、提示

- **添加多个工具**：只需将多个工具实例放入 `tools` 列表，并在 `bind_tools` 中使用，工具节点会自动处理。
- **工具返回复杂数据**：可以使用 `json.dumps` 将结果序列化，或者直接返回字符串。
- **错误处理**：在工具节点中捕获异常，返回友好的错误信息，避免整个对话中断。
- **调试**：打印 `state["messages"]` 可以观察消息流转过程，帮助排查问题。
